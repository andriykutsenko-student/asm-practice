# int 0x80 — системні виклики в Linux (x86)

## Що таке int 0x80?

`int 0x80` — це інструкція переривання в архітектурі x86, яка використовується
для виконання системних викликів (syscalls) в Linux. Коли програма виконує цю
інструкцію, процесор переключається з режиму користувача в режим ядра, і ядро
Linux виконує запитану операцію.

Перед викликом `int 0x80` необхідно заповнити регістри:
- `eax` — номер системного виклику
- `ebx` — перший аргумент
- `ecx` — другий аргумент
- `edx` — третій аргумент
- `esi` — четвертий аргумент
- `edi` — п'ятий аргумент

Результат повертається в регістрі `eax`.

---

## Найбільш часто використовувані системні виклики

### 1. sys_exit (eax = 1)
Завершення програми.

**Параметри:**
- `ebx` — код завершення (0 = успіх)

**Приклад:**
```asm
mov eax, 1
mov ebx, 0
int 0x80
```

---

### 2. sys_write (eax = 4)
Запис даних у файловий дескриптор.

**Параметри:**
- `ebx` — файловий дескриптор (1 = stdout, 2 = stderr)
- `ecx` — адреса буфера з даними
- `edx` — кількість байт для запису

**Приклад:**
```asm
mov eax, 4
mov ebx, 1
mov ecx, message
mov edx, 13
int 0x80
```

---

### 3. sys_read (eax = 3)
Читання даних з файлового дескриптора.

**Параметри:**
- `ebx` — файловий дескриптор (0 = stdin)
- `ecx` — адреса буфера для читання
- `edx` — кількість байт для читання

**Приклад:**
```asm
mov eax, 3
mov ebx, 0
mov ecx, buffer
mov edx, 64
int 0x80
```

---

### 4. sys_open (eax = 5)
Відкриття файлу.

**Параметри:**
- `ebx` — адреса рядка з іменем файлу
- `ecx` — прапори (0 = O_RDONLY, 1 = O_WRONLY, 2 = O_RDWR)
- `edx` — права доступу (наприклад, 0644)

**Приклад:**
```asm
mov eax, 5
mov ebx, filename
mov ecx, 0
mov edx, 0644
int 0x80
```

---

### 5. sys_close (eax = 6)
Закриття файлового дескриптора.

**Параметри:**
- `ebx` — файловий дескриптор

**Приклад:**
```asm
mov eax, 6
mov ebx, 3
int 0x80
```

---

### 6. sys_fork (eax = 2)
Створення дочірнього процесу.

**Параметри:** немає

**Приклад:**
```asm
mov eax, 2
int 0x80
```

---

### 7. sys_getpid (eax = 20)
Отримання ідентифікатора поточного процесу.

**Параметри:** немає

**Приклад:**
```asm
mov eax, 20
int 0x80
```
